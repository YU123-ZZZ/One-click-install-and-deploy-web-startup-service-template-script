一个面向 Windows 的纯 CMD 中文版服务管理模板。它把“检查环境、配置启动命令、处理端口、启动、停止、重启、查看状态”集中到同一个菜单，适合需要反复在本机或局域网环境启动 Java、Node.js、Python 等服务的人。

它不是复杂的运维平台，也不试图替代 Docker、NSSM、Windows 服务或 CI/CD。它的目标很明确：让一个原本需要记住多条命令、手动找端口和反复排查环境的问题，变成可见、可重复、可恢复的本地操作流程。

## 你可能正遇到的问题

| 常见场景 | 原来的麻烦 | 模板如何处理 |
| --- | --- | --- |
| 每次启动都要重新输入一长串命令 | 参数容易漏，端口经常写错 | 保存启动命令、文件名、附加参数和端口配置 |
| 程序启动失败但不知道缺什么 | Java、Node.js、Git 是否可用要逐项查 | 一次检查所需运行环境，并可调用 winget 安装 Java、Node.js、Git |
| 默认端口已被占用 | 只能手动用 netstat 查找并换端口 | 检测端口；一键部署时可自动向后寻找可用端口 |
| 启动后关闭 CMD，服务也被关掉 | 管理窗口和服务进程绑定 | 服务以独立最小化窗口运行，关闭管理窗口后继续运行 |
| 下次打开脚本看不到之前服务 | 不知道服务是不是仍在运行 | 保存端口和 PID；下次打开时双重校验并恢复状态 |
| 修改配置后不确定是否生效 | 需要手动停止、再复制命令重启 | 保存后会询问是否立即重启并应用新配置 |
| 误杀了其他项目的进程 | 只根据端口判断不够可靠 | 仅在端口和保存的 PID 同时匹配时接管或停止本模板服务 |

## 为什么选择我这个模板

### 1. 有效解决最影响效率的问题

模板优先处理本地部署中最频繁、最容易出错的环节：环境、命令、端口、进程和状态。它不要求你先学习新的运维工具，也不要求改造项目结构。已有项目只要能用一条前台命令启动并监听 TCP 端口，就可以接入。

纯中文，比较友好。

### 2. 纯 CMD，文件少，迁移成本低

核心是一个 `.cmd` 文件。把它放到项目目录，填一次配置即可复用；没有额外运行时、没有隐藏下载器，也没有 PowerShell、VBS、MSHTA 或图形化提权逻辑。

### 3. 操作透明，不自动越权

脚本不会自动申请管理员权限。只有安装环境、增加或删除防火墙规则时，才明确提示你使用“以管理员身份运行”。这比静默提权更容易审计，也更适合在个人电脑、实验室和局域网机器上使用。

### 4. 不只是启动，更重视可恢复管理

启动时会写入 `service.state`，保存端口和 PID。再次打开脚本时，模板会检查端口是否仍在监听、监听 PID 是否仍与记录一致。这样既能继续管理还在运行的服务，也能降低误把别的程序当成当前项目的风险。

### 5. 给新手一个清晰入口，也给熟手保留控制权

菜单把常用操作集中起来；同时启动命令、前置参数、程序文件、附加参数、端口参数和环境依赖都能配置。你不需要被固定在 Spring Boot 默认写法中。

## 核心能力

- 一键部署启动、默认端口启动、自定义端口启动；
- 停止、重启和状态查看；
- 端口检测、常用端口查看、结束占用进程；
- 防火墙端口规则的添加和删除；
- Java、Node.js、Git、winget 环境检查；
- 通过 winget 安装 Java、Node.js、Git；
- 项目配置保存到 `service.conf`；
- 服务端口和 PID 保存到 `service.state`；
- 删除已保存的环境配置并恢复默认参数；
- 彩色输出：标题、成功、提示和错误状态一眼可分辨；
- `在线演示.html`：与 CMD 菜单一致的浏览器模拟界面，不会执行任何系统操作。

## 受众

- 需要在 Windows 上频繁启动本地后端服务的开发者；
- 需要把项目交给同事、同学或测试人员运行，但不想让对方记复杂命令的人；
- 使用 Java JAR、Node.js 入口文件、Python 入口文件等单命令启动项目的人；
- 需要在局域网中临时开放一个服务，并希望快速确认端口和访问地址的人；
- 希望先用轻量脚本管理本地服务，而不是立即引入完整部署系统的人。

## 不适合的场景

请优先使用更专业的工具，而不是勉强套用本模板：

- 需要开机自启动、异常自动拉起、服务账号隔离：使用 Windows 服务、NSSM 或任务计划程序；
- 需要容器化、镜像分发、多服务编排：使用 Docker Compose、Kubernetes 等；
- 需要远程批量发布、审批、日志聚合或持续集成：使用 CI/CD 和专业运维平台；
- 项目无法通过一条前台命令启动，或不监听 TCP 端口：需要先调整项目启动方式。

## 文件说明

| 文件 | 作用 |
| --- | --- |
| `一键安装部署启动服务模板.cmd` | 真正执行环境检查、部署与服务管理的脚本 |
| `在线演示.html` | CMD 界面的浏览器模拟环境，不执行本机命令 |
| `README.md` | 使用、迁移与注意事项说明 |
| `service.conf` | 保存项目名称、启动命令、程序文件、端口和依赖；首次保存配置后生成 |
| `service.state` | 保存当前服务端口和 PID；启动成功后生成 |

## 快速开始

1. 将 `一键安装部署启动服务模板.cmd` 放到项目入口文件所在目录；
2. 双击脚本；
3. 进入“项目管理 → 修改配置”；
4. 根据项目原始启动命令填写参数；
5. 输入 `S` 保存；
6. 先选择“使用默认端口启动”验证；
7. 确认能访问后，再使用“一键部署启动”。

普通启动、停止、重启和状态查看不需要管理员权限。安装环境、添加或删除防火墙规则时，请右键 CMD 选择“以管理员身份运行”。脚本不会自动提权。

## 接入其他项目

### 第一步：先确认原始命令能运行

不要先改脚本。请先在普通 CMD 中运行项目原始命令，并确认它可以正常监听端口。例如：

```text
java -jar app.jar --server.port=8080
node server.js --port=3000
python app.py --port=5000
```

原始命令不能成功，模板也无法替代项目本身排错。

### 第二步：在“修改配置”中拆分命令

脚本最终按照下面结构启动项目：

```text
启动命令 命令前参数 "程序文件" 启动附加参数 端口参数+端口号
```

| 配置项 | 应填写什么 | 示例 |
| --- | --- | --- |
| 项目名称 | 菜单里显示的名字 | `订单后台` |
| 默认端口 | 项目要监听的端口 | `8080` |
| 启动命令 | 实际运行程序 | `java`、`node`、`python` |
| 命令前参数 | 程序文件前面的参数 | Java 使用 `-jar` |
| 启动附加参数 | 程序文件后面的固定参数 | `--spring.profiles.active=prod` |
| 程序文件 | 项目入口文件 | `app.jar`、`server.js`、`app.py` |
| 端口参数 | 端口号之前的参数前缀 | `--server.port=`、`--port=` |
| 环境依赖 | 是否检查 Java、Node.js、Git | `1` 或 `0` |

### Java / Spring Boot 示例

```text
启动命令：java
命令前参数：-jar
程序文件：app.jar
启动附加参数：留空
端口参数：--server.port=
需要 Java：1
需要 Node.js：0
```

对应命令：

```text
java -jar "app.jar" --server.port=8080
```

### Node.js 示例

前提：项目支持 `--port=端口号`。

```text
启动命令：node
命令前参数：输入减号清空
程序文件：server.js
启动附加参数：留空
端口参数：--port=
需要 Java：0
需要 Node.js：1
```

如果项目只读取环境变量 `PORT`，或使用 `npm run` 作为唯一启动方式，需要先让项目支持命令行端口参数，或使用与模板配置一致的固定端口。

### Python 示例

前提：项目支持 `--port=端口号`。

```text
启动命令：python
命令前参数：输入减号清空
程序文件：app.py
启动附加参数：留空
端口参数：--port=
需要 Java：0
需要 Node.js：0
```

Python 不在模板的自动安装列表中，请自行安装，并确保 `python --version` 可正常执行。

## 服务关闭、再次打开与修改配置

服务启动成功后，项目会在独立的最小化窗口中运行。直接关闭管理脚本窗口，服务会继续运行。

下次再打开 CMD 时，脚本会读取 `service.state` 并验证端口与 PID：

- 两者一致：显示为“运行中”，可以继续停止、重启或修改配置；
- 端口没有监听或 PID 不一致：自动清理旧状态，避免误管其他服务；
- 保存新配置时：如果服务正在运行，脚本会询问是否立即重启并应用新配置。

注意：它不是 Windows 服务。电脑重启、服务窗口被关闭或进程异常退出后，不会自动启动。

## 删除环境配置

进入“项目管理 → 删除环境配置”，输入 `Y` 后会删除 `service.conf`，并将项目名称、端口、启动命令和依赖项恢复为默认模板值。

该操作不会停止当前运行中的服务，也不会删除 `service.state`。如果要彻底切换项目，建议按以下顺序操作：

1. 在主菜单停止当前服务；
2. 删除旧的 `service.state`；
3. 使用“删除环境配置”；
4. 把模板放入新项目目录后重新配置。

## 安全与使用边界

- `service.conf` 是可执行的 CMD 配置文件，只使用自己创建或确认可信的文件；
- 首次处理端口占用时，先确认 PID 所属程序，不要盲目结束未知进程；
- 防火墙规则会影响本机网络暴露范围，仅放行确实需要对局域网开放的端口；
- 端口参数必须是项目实际支持的格式；不支持时可清空端口参数，但项目监听端口必须与模板配置一致；
- 推荐在 Windows 10/11 的 Windows Terminal 或新版控制台中使用；
- `在线演示.html` 只展示交互流程，不能代替 CMD 真正部署服务。

## 颜色说明

- 青色：标题和当前菜单；
- 绿色：成功、已完成或运行中；
- 黄色：提示、警告或未运行；
- 红色：错误或执行失败；
- 白色：普通信息。
